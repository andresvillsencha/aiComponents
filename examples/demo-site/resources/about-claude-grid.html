Healthcare Grid with Natural-Language Integration

This example demonstrates how to use a custom natural-language (NL) search component within an Ext JS grid displaying healthcare shift data.

The user can type natural-language queries in the search custom component located in the grid toolbar to filter, sort, group, navigate pages, or even show and hide columns.

The component connects to a backend NodeJS NL service powered by OpenAI, which interprets the user's intent and returns a structured JSON containing filters, sorters, grouping, and pagination instructions.

Usage

By using the NL search bar, users can type queries in plain English to explore shift data. Examples include:

- Show nurses on night shift
- Show shifts longer than 8 hours
- Show cardiology department only
- Sort by date ascending
- Sort by shift duration descending
- Group by department
- Show all active employees
- Go to next page
- Change page size to 100
- Hide all columns
- Show column Letter

Files

Custom Component: ux.ai.SmartSearch

  • /app/ux/ai/SearchBar.js
    Custom component that gathers column information from the linked grid. It sends the user prompt and the column data to the backend. Using that information, the backend generates a prompt to be sent to Open AI's Api.

Frontend (Ext JS)

  • /app/view/ai/DataGrid.js
    Healthcare Grid view: columns, toolbar with NL input and standard column filters, binds to the store.
  • /app/view/ai/DataGridController.js
    View Controller that is used to process the view's events and user interactions, like opening the view response window.
  • /app/store/BankingDataStore.js
    Client-side store that load sales rows from the backend. The store supports local filtering and sorting; data is loaded from the server after just once, and is filtered locally every user query.

Backend (Node.js)

  • Data Server

    • /server/server.js
      Express bootstrap: serves static assets and exposes the NL endpoint used by the grid controller.
    • /server/endpoints/banking-endpoint.js
      Builds the NL prompt from the user text, calls the NL model API, and returns the model's structured interpretation (not raw rows).     • /server/endpoints/banking-data.js

    • /server/server.js

      Endpoint handler. Receives the request from the EXT JS component, and calls the correct Node JS file depending on the specified provider (chatgpt or claude).

      Builds the NL prompt from the user text, calls the NL model API, and returns the model's structured interpretation (not raw rows).
    • /server/endpoints/ai/services/claude.js
      Builds the NL prompt from the user text, calls the NL model API, and returns the model's structured interpretation (not raw rows).

Typical runtime flow (step-by-step)

  1. User types a free-text query in the Searchbar inside the grid toolbar.
  2. The custom component gathers the grid's columns and their filters.
  3. The custom component sends the text and column data to the backend NL endpoint (POST /api/ai-endpoint).
  4. ai-endpoint.js composes a prompt automatically and calls the NL model. The model returns a structured interpretation.
  5. The custom component receives the response and applies filters to the filters plugin, sorters to the store, and grouping and paging to the grid.

The response

Once the middleware responds, it returns a JSON object that can include up to four main sections: filters, sorters, grouping, and tokenData.

  • filters
    Contains the filtering conditions generated from the user's query, defining which records should be displayed. Ready to be applied to the grid filters.
  • sorters
    Specifies how the data should be ordered (e.g., by field and direction such as ascending or descending) ready to be applied to the store.
  • grouping
    Describes how records are grouped by a specific field.
  • paging
    Allows page navigation and size.
  • columns
    Describes the visibility of columns.
  • tokenData
    Provides the total tokens consumed during the request.

About the custom component

The AI Search Bar is a custom Ext JS component designed to bring natural-language search to data grids. Instead of using traditional filter menus, users can type plain-English queries (for example, “show nurses on night shift”) directly into the search bar.

When a query is submitted, the component collects information about the grid — such as its columns and active filters — and sends that context to a backend service. The backend then interprets the request (through an AI model) and returns structured instructions on how the grid should be filtered, sorted, grouped, or paged.

This component doesn't talk directly to the AI model; it relies on a middleware service to handle communication with OpenAI or other natural-language engines. It’s designed to sit inside a grid's toolbar (or be linked to a specific grid) so it can automatically apply the AI's instructions to the grid's data.

In short, Ext.ai.SmartSearch lets users interact with complex data through simple language, transforming natural sentences into dynamic grid actions.

Although the provided middleware was created with NodeJS, it can be any backend of your preference.
