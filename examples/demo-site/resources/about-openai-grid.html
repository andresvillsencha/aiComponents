Smart Search Component with OpenAI

This example demonstrates how to use the Smart Search Component within an Ext JS grid displaying school data.

The user can type natural-language queries in the component located in the grid toolbar to perform different accions: filter, sort, group, navigate pages, and/or show and hide columns.

The component connects to a backend NodeJS Middleware service powered by OpenAI, which interprets the user's intent and returns a structured JSON containing filters, sorters, grouping, columns, and pagination instructions.

Usage

By using the Smart Search Component, users can type queries in plain English to explore data. Examples include:

- Show students on 8 and 9 grade
- Students with an average score between 70 and 80
- Show A grade students
- Sort by status
- Sort by gender and age
- Group by letter grade
- Go to first page
- Go to next page
- Change page size to 300
- Show 1000 rows per page
- Hide all columns but name and grade
- Show column Letter
- Reset grid

Files

Package

  • /packages/ai-components/
    Ext JS Custom Components Package that connects to a Node JS middleware It automatically gathers field or column information and sends it to the NodeJS middleware along with the user prompt. The middleware generates a prompt to be sent to Open AI's Api.

Middleware

The middleware service was implemented using NodeJS. No code needs to be modified; However, the developer needs to do the following:

  • Init NodeJS App.
  • Install ai-server files.
  • Create .env file based on .env.dev
  • Add API key, port, and CORS exceptions into the .env file.
  • If necessary, the system-prompt file can be modified, or a new file can be created.
  • Run the server in an available port.

The middleware host the AI model API key, making it impossible for users to grab the key and use it for another purpose. Having the middleware server also allows to protect from non-authorized calls by using CORS validation.

Typical runtime flow (step-by-step)

  1. User types a free-text query in the SmartSearch component.
  2. The component gathers the grid's columns and their filters.
  3. The component sends the text and column data to the middleware NodeJS endpoint (POST /api/ai-smart-search).
  4. The ai-smart-search api creates a prompt automatically and calls the OpenAI API. Returning a structured json interpretation.
  5. The ExtJS SmartSearch component receives the response and applies it to the filters plugin, the store, or to the grid.

The response

Once the middleware responds, it returns a JSON object that can include up to following main sections:

  • filters
    Contains the filtering conditions generated from the user's query, defining which records should be displayed. Ready to be applied to the grid filters.
  • sorters
    Specifies how the data should be ordered (e.g., by field and direction such as ascending or descending) ready to be applied to the store.
  • grouping
    Describes how records are grouped by a specific field.
  • paging
    Allows page navigation and size.
  • columns
    Describes the visibility of columns.
  • clear
    Indicates features to clear. For example, to clear grouping.
  • tokenData
    Provides the total tokens consumed during the request.

About the custom component

The Smart Search Component is designed to bring natural-language search to data grids. Instead of using traditional filter menus, users can type plain-English queries (for example, “show students with an average grade of 80 or higher”) directly into the search bar.

When a query is submitted, the component collects information about the grid — such as its columns and active filters — and sends that context to a middleware service. The middleware then interprets the request (through an AI model) and returns structured instructions on how the grid should be filtered, sorted, grouped, or paged.

This component doesn't talk directly to the AI model; it relies on a middleware service to handle communication with OpenAI or other natural-language engines. It's designed to sit inside a grid's toolbar (or be linked to a specific grid) so it can automatically apply the AI's instructions to the grid's data.

In short, Ext.ai.SmartSearch lets users interact with complex data through simple language, transforming natural sentences into dynamic grid actions.

Although the provided middleware was created with NodeJS, developers are free to use any backend of your preference by creating their own middleware server.
