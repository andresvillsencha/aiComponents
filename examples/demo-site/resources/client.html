Setting Up Ext JS and Running the Frontend

To start, you need an existing Sencha Ext JS 7+ Application or creating a new one. For more information, visit the Sencha Documentation.

File Description

DataGrid.js
  • Defines the view
  • Includes a toolbar with the search bar
  • Enables plugins/features you plan to use (e.g., gridfilters, grouping, pagingtoolbar)
  • Binds to DataStore
DataGridController.js
  • In grids where there is no custom component:
    • Listens to events from the NL SearchBar callback
    • Applies filters/sorters/group instructions (and triggers store loads when needed)
  • In all grids:
    • Opens the response window
    • Clears filters
DataStore.js
  • Configures the proxy to your backend server or json data
  • Sets pageSize and remoteFilter/remoteSort as needed
  • Defines fields that match the grid columns
ux/ai/SearchBar.js To be used in grids with the custom AI component.
  • Your Ext.ai.SmartSearch class (provided component)
  • Make sure serverUrl and endpoint point to your Node server (e.g., http://localhost:3000, /api/ai-endpoint)
  • Must be placed in a grid toolbar or linked to a grid via the grid config

The custom ux.ai.SmartSearch Component

When a user enters a natural-language query, the component first captures the user's prompt from the search field. It then automatically extracts the column definitions and any active filter metadata from the linked grid to provide context for the backend. This combined information is sent as a POST request to the backend endpoint, where the natural-language interpretation occurs. Once the response is received, the component processes it and applies the returned filters, sorters, grouping, and page navigation instructions directly to the grid, updating the displayed data accordingly.

  • Capture the user prompt
  • Extract column + filter metadata from the linked grid
  • POST to the backend
  • Apply the returned filters/sorters/group/page to the grid
