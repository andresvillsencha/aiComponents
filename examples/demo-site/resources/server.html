Setting Up Node.js and Project Dependencies

Install Node.js

Download and install Node.js from NodeJS.org. Once installed, verify it's correct installation using your terminal or command prompt window:

node -v 
npm -v

This confirms Node.js and npm (Node Package Manager) are working.

Create a New Project

Create a new folder for your project, navigate into the folder where your middleware service will be hosted, and initialize it. For example:

mkdir my-project
cd my-project
npm init -y

The -y flag automatically creates a default package.json file.

Install Required Packages

Use npm to install the dependencies you'll need:

npm install express
npm install cors
npm install dotenv
npm install body-parser
npm install openai
npm install @anthropic-ai/sdk
npm install axios

Here's what each package does:

  • express: for building the web server and API endpoints
  • cors: enables cross-origin requests (useful when connecting frontend & backend)
  • dotenv: loads environment variables from a .env file
  • body-parser: parses incoming JSON data in requests
  • openai: allows your Node.js app to access OpenAI's API
  • anthropic: allows your Node.js app to access Anthropic/Claude's API.
  • axios: allows your Node.js app to access your local LLM API

Set Up the .env File

Create a file named .env in the root of your project and add your OpenAI API key:

OPENAI_API_KEY=your_openai_api_key_here
CLAUDE_API_KEY=your_claude_api_key_here

You must also indicate the server port and the allowed CORS domains:

PORT=3001
CORS=comma_separated_list_of_allowed_urls_must_include_http

If required, add a default model, and a list of available models:

VALIDATE_MODEL=true
OPENAI_DEFAULT_MODEL=gpt-4o-mini
OPENAI_MODELS=gpt-4o-mini,gpt-4-turbo,gpt-4,gpt-4o
CLAUDE_DEFAULT_MODEL=claude-sonnet-4-20250514
CLAUDE_MODELS=claude-sonnet-4,claude-sonnet-4-20250514

File Structure

This example is composed of several backend files. It includes:

  • The public AI-prompt handler, which exposes the endpoint the frontend application will be calling.
  • Service modules responsible for connecting to your chosen AI provider.
  • The prompt folder with JSON files defining the AI system prompt.
  • The env variable configuration file.
Files related to data retrieval are not included, since those depend entirely on your own backend architecture and business logic.

ai-server
  • endpoints
    • ai
      • handlers
        • ai-smart-fill.js
        • ai-smart-search.js
      • prompts
        • system-prompt.json
        • form-prompt.json
        • your-own-custom-prompt.json
      • services
        • ...
  • .env
  • ai-server.js

Once you have everything ready, run the following command to start the server: 

node ai-server.js
